| 4 | R0 落地计划：XML Profile 示例 + QoS 量化实验设计 |

### 通信拓扑核心

```
Sensor  → Topic (best_effort/reliable) → Fusion
Fusion  → Topic (reliable)             → Decision
Decision → Action (Goal/Feedback/Result) → Motor
External → Service (req/rsp)           → Motor
```

### 分层约束

每层有明确的 "允许依赖" 和 "禁止依赖"，接口层不依赖任何节点实现。

## 今日关键认知

1. **工程标准要抽象到原则级**，不绑定具体文件名。Clean Code / Google Style / ROS2 惯例是三套并列规范，不是优先级链

2. **PRD 要先定"不做"**：明确 ~R11~ ~R12~ 才能聚焦核心 R0-R10

3. **DDS 层能力是核心技术深度**：R0 的 XML Profile + QoS 量化实验是深入理解 DDS 的关键

4. **Fast-DDS 不是独立一层**，它贯穿所有通信箭头。架构图不该画 DDS 框，它是隐式存在的通信基础设施

5. **接口契约评审要逐条过**：.msg/.srv/.action 一但确定就是 6 个节点的契约，改起来成本高
