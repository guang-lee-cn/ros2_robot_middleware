# 自变量机器人 技术一面 — 卡尔曼滤波

## 高频问题

### Q1: 你实现的 KF 是线性的还是扩展的（EKF）？为什么不选 EKF？

**考点：** 理解线性 vs 非线性状态转移的区别

**30 秒回答：**
> 我实现的 4 状态（x, y, vx, vy）恒加速度模型是线性的——状态转移 `x_{k+1} = x_k + vx*dt + 0.5*ax*dt²` 不含三角函数和叉积。EKF 只有在模型非线性（比如涉及航向角 θ 的 cos/sin）时才需要。我在这个场景选择线性 KF 是因为恒加速度在短时间窗口内足够准确，而且不需要计算 Jacobians。robot_localization 的 EKF 虽然更通用，但对这个 demo 来说是大炮打蚊子。

---

### Q2: 五个矩阵 F/P/Q/H/R 分别是什么意思？

**考点：** KF 核心概念的精确理解

| 矩阵 | 名称 | 维度 | 含义 |
|------|------|------|------|
| **F** | 状态转移矩阵 | 4×4 | 描述系统如何从 `t` 演变到 `t+1`。恒加速度模型：位置 += 速度×dt，速度不变 |
| **P** | 协方差矩阵 | 4×4 | 状态估计的不确定性。对角大 = 不确信，对角小 = 确信。预测步增大（变不确定），更新步缩小（变确定） |
| **Q** | 过程噪声协方差 | 4×4 | 模型预测本身的不确定性。Q 大 = 模型不准，更多信任测量。我的 x,y 噪声 0.01，v 噪声 0.1——位置假设比速度更可靠 |
| **H** | 测量矩阵 | 2×4 | 把状态空间映射到测量空间。H=[1,0,0,0; 0,1,0,0] 意思是"我只能测量位置，测不到速度" |
| **R** | 测量噪声协方差 | 2×2 | 传感器噪声。我的 R=0.1 表示 LiDAR 测距噪声约 ±0.1m。R 大 = 传感器不准，更多信任模型 |

**面试追问："如果 LiDAR 突然变准了（比如换了高精度传感器），你改哪个参数？"** → R，减小测量噪声。

**面试追问："如果机器人在冰面上打滑，IMU 的数据不可靠，改哪个？"** → Q，增大过程噪声（模型预测更不可信）。

---

### Q3: 你的 KF 更新步把 x 和 y 分开独立更新，这样对吗？什么时候需要联合更新？

**考点：** 测量噪声独立 vs 相关

**答案：**
> 分开独立更新 x 和 y 等价于假设测量噪声协方差 R 是对角矩阵（x 的误差和 y 的误差不相关）。如果 LiDAR 的测距误差在 x 和 y 方向独立（大多数场景满足），这没问题。但如果你的传感器是一个旋转的雷达，笛卡尔坐标 (x,y) 的噪声其实是从极坐标 (r,θ) 非线性转换过来的，x 和 y 的误差相关，就应该做完整的 2D 联合更新。

---

### Q4: 你的 dt 写死 0.2。如果 FusionNode 的 timer 不准（比如系统负载高，间隔变成 0.3），KF 会怎样？

**考点：** 实时系统中的时间敏感性

**答案：**
> 用固定 dt=0.2 在 timer 不准时会引入额外误差。正确做法是记录上一次 predict 的时间戳，每次计算实际的 `dt = now - last_predict`。vx*dt 对于 0.2→0.3 的偏差会产生 50% 的位置估计误差。这是 M3.3 Executor 优化要解决的一个点——保证 timer 回调及时执行。

---

### Q5: 你的 KF 线程安全吗？timer_callback 在哪个线程执行？

**考点：** ROS2 回调线程模型 + 数据竞争

**答案：**
> 不线程安全——但当前不需要。timer_callback 和 sensor callbacks 都在同一个 SingleThreadedExecutor 的线程里顺序执行（spin_once 保证同一时刻只有一个回调在跑）。如果 M3.3 换成 MultiThreadedExecutor，sensor callback 和 timer callback 可能并发访问 kf_ 的 state_ 和 cov_，就需要加 mutex。
>
> 当前选择单线程不是偷懒，是可控的复杂度管理——多线程+KF 的调试成本远大于单线程的吞吐瓶颈。

## 代码手撕环节

面试官可能让你在白板上写 KF predict 步骤的伪代码。标准答案：

```
function predict(x, P, u, dt):
    F = [[1, 0, dt, 0],   // 状态转移
         [0, 1, 0,  dt],
         [0, 0, 1,  0],
         [0, 0, 0,  1]]
    B = [[0.5*dt², 0],     // 控制矩阵
         [0,       0.5*dt²],
         [dt,      0],
         [0,       dt]]
    
    x = F·x + B·u          // u = [ax, ay] from IMU
    P = F·P·F^T + Q        // 协方差传播 + 过程噪声
    return x, P
```
